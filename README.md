# telecom-churn-prediction
Цей проєкт присвячений розробці ML-рішення для прогнозування відтоку клієнтів (Customer Churn) у телекомунікаційній компанії.
Метою є визначення клієнтів з високою ймовірністю відтоку, щоб компанія могла завчасно застосувати retention-стратегії (знижки, персональні пропозиції, покращення сервісу тощо).
Проєкт реалізований у Python, має інтерактивний вебінтерфейс на Streamlit та розгортається за допомогою Docker.
Задача проєкту

Розв’язується задача прогнозування відтоку клієнтів (Customer Churn Prediction) для телекомунікаційної компанії.

Тип задачі: бінарна класифікація
Цільова змінна: churn
    1 — клієнт піде
    0 — клієнт залишиться

Дані: Використовуються історичні дані клієнтів телекомунікаційної компанії.
Встановлення та запуск (Docker):
    Вимоги: Docker
            Docker Compose
    Запуск проєкту:
            У корені проєкту виконайте: docker-compose up --build
            Після успішного запуску застосунок буде доступний за адресою: http://localhost:8501
Після загрузки додатку, для прогнозування відтоку клієнта, в додатку передбачене введення наступних даних:
    Наявність терміну контракту,
    Remaining contract (years),
    TV subscriber,
    Movie package,
    Subscription age,
    Bill avg,
    Service failures,
    Download avg,
    Upload avg,
    Over limit

Далі передбачен вибір однієї з чотирьох моделей прогнозуванн відтоку:
    -Logistic Regression (інтерпретована та зручна для бізнесу)
    -Random Forest
    -Gradient Boosting
    -Neural Network (MLP)
після натискання кнопки "Спрогнозувати" буде надано результат:
в вигляді % ймовірності відтоку та таблиці, в якій представлено метрики вибраної моделі:
    =Accuracy,
    -Precision,
    -Recall,
    -F1-score
а також Classification Report.
